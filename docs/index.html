One Logic. Every Platform.

NotifyZen is the definitive bridge for Firebase Cloud Messaging. We have eliminated the friction of device fingerprints, platform detection, and background interaction tracking, providing a single TypeScript API that works across your entire stack.

Optimized Setup

Install the core engine and the required fingerprinting modules.

Web Environments

npm install notifyzen-every-where @fingerprintjs/fingerprintjs

Mobile Native Environments

npm install @react-native-firebase/messaging react-native-device-info

⚛️ React Implementation

Our useNotifyZen hook provides a stateless, clean reactive interface for your components.

Complete Logic Component

import React, { useEffect, useState } from 'react';
import { useNotifyZen } from 'notifyzen-every-where';

const credentials = { /* Your FCM Web Config */ };

export const NotificationApp = () => {
  const [messages, setMessages] = useState([]);

  // 1. Initialize with your secret and topics
  const { 
    onNotification, 
    onNotificationClick, 
    currentToken, 
    isInitializing 
  } = useNotifyZen(credentials, {
    secretKey: "your_api_secrate",
    topics: ["news", "offers"],
    debug: true
  });

  // 2. Set up unified listeners
  useEffect(() => {
    // 🔔 Arrival Listener (Foreground)
    const unsubMsg = onNotification((n) => {
       setMessages(prev => [n, ...prev]);
    });

    // 🖱️ Interaction Listener (Click/Tap)
    const unsubClick = onNotificationClick((n) => {
       console.log("User tapped notification:", n.id);
       router.push(n.data.url); // Navigate smoothly
    });

    return () => { unsubMsg(); unsubClick(); };
  }, [onNotification, onNotificationClick]);

  if (isInitializing) return ;

  return (
    
  );
};

⚡️ Next.js Framework Strategy

Next.js requires careful handling of Server-Side Rendering (SSR). Our engine is SSR-safe by default, but requires client-side activation.

The 'Use Client' Pattern

/**

    <>
      {children}

📱 Native Mobile Deployment

Bridge the native Firebase dependencies into our universal logic flow.

Bridging Native Dependencies

import messaging from '@react-native-firebase/messaging';
import { useNotifyZen, createNativeProvider } from 'notifyzen-every-where';

// 1. Create the Bridge Instance
const nativeProvider = createNativeProvider(messaging());

export default function NativeApp() {
  // 2. The API remains exactly the same as Web!
  const { onNotificationClick } = useNotifyZen(credentials, {
    secretKey: "your_mobile_secrate",
    provider: nativeProvider, // Inject the native bridge
    topics: ["ios_user", "mobile_global"],
    debug: true,
  });

  useEffect(() => {
    // 3. Automated support for Background & Quit-state clicks
    const unsub = onNotificationClick((n) => {
      alert('Opened from notification: ' + n.title);
    });
    return unsub;
  }, [onNotificationClick]);

  return ;
}

🛠️ Troubleshooting Guide

Expert solutions for the most common integration hurdles.

Issue: "Mobile Device ID fallback generated"

Ensure you have react-native-device-info installed and linked. In iOS, you MUST run npx pod-install to activate the hardware identification modules.

Issue: "Failed to load Web Firebase Messaging"

This happens if window is not available or if the Firebase scripts are blocked by an AdBlocker. The library handles this gracefully, but check your browser console for CSP errors.
